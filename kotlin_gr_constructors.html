Κατασκευαστές

Σε άλλο σημείο του οδηγού εκμάθησης δημιουργήσαμε αντικείμενο μίας κλάσης και προσδιορίσαμε τις ιδιότητές του μέσα στην κλάση, ως εξής: 

class Car {
    var brand = ""
    var model = ""
    var  year = 0
}

fun main() {
    val c1 = Car()
    c1.brand = "Ford"
    c1.model = "Mustang"
    c1.year = 1969
}

Στην Kotlin υπάρχει ταχύτερος τρόπος να το κάνουμε, με τη χρήση ενός κατασκευαστή (constructor).

Ένας κατασκευαστής είναι κάτι σαν ειδικού τύπου συνάρτηση (special function), και ορίζεται με τη χρήση δύο παρενθέσεων () μετά από το όνομα της κλάσης. Μπορούμε να καθορίσουμε τις ιδιότητες μέσα στις παρενθέσεις (όπως περνούμε παραμέτρους μέσα σε μία κανονική σνάρτηση).

Ο κατασκευαστής θα αρχικοποιήσει τις ιδιότητες μόλις δημιουργήσουεμ ένα αντικείμενο της κλάσης. Απλώς να θυμόμαστε να καθορίσουμε τον τύπο της ιδιότητας/μεταβλητής: 

class Car(var brand: String, var model: String, var year: Int)
    fun main() {
        val c1 = Car("Ford", "Mustang", 1969)
}

Τώρα είναι ακόμη ευκολότερο να καθορίσουμε πολλαπλά αντικέιμενα σε μία κλάση: 

class Car(var brand: String, var model: String, var year: Int)

                fun main() {
    val c1 = Car("Ford", "Mustang", 1969)
    val c2 = Car("BMW", "X5", 1999)
    val c3 = Car("Tesla", "Model S", 2020)
}

Μία κλάση στην Kotlin μπορεί να έχει ένα πρωτεύοντα κατασκευαστή (primary constructor) και έναν ή περισσότερους δευτερεύοντες κατασκευασές (secondary constructors). Ο πρωτεύων κατασκευαστής αποτελεί τμήμα του class header και πηγαίνει μετά από το όνομα της κλάσης και τους προαιρετικούς τύπους παραμέτρων. 

class Person constructor(firstName: String) { /*...*/ }

Αν ο πρωτεύων κατασκευαστής δεν έχει annotations ή visibility modifiers, το keyword constructor μπορεί να παραλειφθεί: 

class Person(firstName: String) { /*...*/ }

Ο πρωτεύων κατασκευαστής δεν μπορεί να περιέχει καθόλου κώδικα. Ο κώδικας αρχικοποίησης μπορεί να τοποθετηθεί σε initializer blocks με το keyword πρόθημα init .

Κατά την αρχικοποίηση ενός στιγμιοτύπου, τα initializer blocks εκτελούνται με την ίδια σειρά που εμφανίζονται στο σώμα της κλάσης, ένθετα με τα property initializers: 

class InitOrderDemo(name: String) {
    val firstProperty = "First property: $name".also(::println)
    
    init {
        println("First initializer block that prints $name")
    }
    
    val secondProperty = "Second property: ${name.length}".also(::println)
    
    init {
        println("Second initializer block that prints ${name.length}")
    }
}

Οι παράμετροι του πρωτεύοντος κατασκευαστή μπορούν να χρησιμοποιηθούν στα initializer blocks. Μπορούν επίσης να χρηιμοποιηθούν στα property initializers έχοντας δηλωθεί στο σώμα της κλάσης: 

class Customer(name: String) {
    val customerKey = name.uppercase()
}

Η Kotlin έχει περιεκτική σύνταξη για τη δήλωση ιδιοτήτων και την αρχικοποίησή τους από τον πρωτεύοντα κατασκευαστή: 

class Person(val firstName: String, val lastName: String, var age: Int)

Τέτοιες δηλώσεις μπορούν επίσης να περιλαμβάνουν default values ιδιοτήτων κλάσης: 

class Person(val firstName: String, val lastName: String, var isEmployed: Boolean = true)

Μπορούμε να χρησιμοποιήσουμε trailing comma όταν δηλώνουμε ιδιοτητες κλάσης: 

class Person(
    val firstName: String,
    val lastName: String,
    var age: Int, // trailing comma
) { /*...*/ }

Παρόμοια με τις κανονικές ιδιότητες, οι ιδιότητες που δηλώνονται στον πρωτεύοντα κατασκευαστή μπορούν να είναι μεταβλητές (var) ή read-only (val).

Αν ο κατασκευαστής έχει annotations ή visibility modifiers, απαιτείται το keyword constructor και τα modifiers πηγαίνουν πριν από αυτό: 

class Customer public @Inject constructor(name: String) { /*...*/ }

Εδώ μπορείτε να βρείτε περισσότερα για τα visibility modifiers.

Δευτευρεύοντες κατασκευαστές (Secondary constructors)

Μία κλάση μπορεί επίσης να δηλώνει δευτερεύοντες κατασκευαστές (secondary constructors), οι οποίοι έχουν πρόθημα το keyword constructor: 

class Person(val pets: MutableList<Pet> = mutableListOf())

class Pet {
    constructor(owner: Person) {
        owner.pets.add(this) // Προσθέτει αυτό το κατοικίδιο στη λίστα με τα κατοικίδια του ιδιοκτήτη τους
    }
}

Αν η κλάση έχει πρωτεύοντα κατασκευαστή, κάθε δευτερεύων χρειάζεται να ανατεθεί στον πρωτεύοντα κατασκευαστή, είτε άμεσα ή έμμεσα μέσω άλλου(ων) δευτερεύοντα(ων) κατασκευαστή(ών). Η ανάθεση σε άλλον κατασκευαστή της ίδιας κλάσης γίνεται με τη χρήση του keyword this: 

class Person(val name: String) {
    val children: MutableList<Person> = mutableListOf()
    constructor(name: String, parent: Person) : this(name) {
        parent.children.add(this)
    }
}

Ο κώδικας στα initializer blocks ουσιαστικά γίνεται τμήμα του πρωτεύοντα κατασκευαστή. Η ανάθεση στον πρωτεύοντα κατασκευαστή συμβαίνει την στιγμή της προσπέλασης της πρώτης δήλωσης ενός δευτερεύοντα κατασκευαστή, έτσι ο κώδικας σε όλα τα initializer blocks και τα property initializers εκτελείται πριν από το σώμα του δευτερεύοντα κατασκευαστή.

Ακόμα και στην περίπτωση που κλάση δεν έχει δευτερεύοντα κατασκευαστή, η ανάθεση εξακολουθεί να συμβαίνει έμμεσα, και τα initializer blocks εξακολουθούν να εκτελούνται: 

class Constructors {
    init {
        println("Init block")
    }
​
    constructor(i: Int) {
        println("Constructor $i")
    }
}

Αν μία μη αφαιρετική κλάση (non-abstract class) δε δηλώνει καθόλου κατασκευαστές (πρωτεύοντα ή δευτερεύοντες), τότε θα παραχθεί πρωτεύων κατασκευαστής χωρίς ορίσματα. Η ορατότητα του κατασκευαστή αυτού είναι public.

Αν δεν επιθυμούμε να έχουμε public κατασκευαστή στην κλάση, δηλώνουμε έναν άδειο πρωτεύοντα κατασκευαστή με non-default visibility: 

class DontCreateMe private constructor() { /*...*/ }

Παρατήρηση: Στη JVM, αν όλες οι παράμετροι του πρωτεύοντα κατασκευαστή έχουν default values, ο compiler θα παράγει έναν επιπλέον κατασκευαστή χωρίς παραμέτρους που θα χρησιμοποιεί τις default values. Κάτι τέτοιο κάνει ευκολότερη τη χρήση της Kotlin με βιβλιοθήκες όπως οι Jackson ή JPA που δημιουργούν στιγμιότυπα κλάσης μέσω κατασκευαστών χωρίς παραμέτρους. 

class Customer(val customerName: String = "")

Δημιουργία στιγμιότυπων κάσης

Για να δημιουργήσουμε στιμιότυπο κλάσης, καλούμε τον κατασκευαστή όπως θα καλούσαμε μία κανονική συνάρτηση: 

val invoice = Invoice()

val customer = Customer("Joe Smith")

Σημείωση: Η Kotlin δε διαθέτει το keyword new.

Η διαδικασία δημιουργίας στιγμιοτύπων σε εμφωλευμένες (nested), εσωτερικές (inner) και ανώνυμες εσωτερικές κλάσεις (anonymous inner classes) περιγράφεται στο λήμμα Nested classes.

Μέλη κλάσης

Οι κλάσεις μπορούν να περιλαμβάνουν:

  • Κατασκευαστές (Constructors) και initializer blocks
  • Συναρτήσεις (Functions)
  • Ιδιότητες (Properties)
  • Εμφωλευμένες (Nested) και εσωτερικές (inner) κλάσεις
  • Δηλώσεις αντικειμένων (Object declarations)

Κληρονομικότητα

Οι κλάσεις μπορούν να παράγονται από άλλες και να σχηματίζουν ιεραρχίες κληρονομικόττας (inheritance hierarchies). Δείτε εδώ περισσότερα στοιχεία για την κληρονομικότητα στην Kotlin.

Αφαιρετικές κλάσεις (Abstract classes)

Μία κλάση μπορεί να έχει δηλωθεί ως abstract, μαζί με κάποια ή όλα τα μέλη της. Ένα abstract μέλος δεν υλοποιείται στην κλάση του. Δε χρειάζεται να σημειώσουμε (annotate) τις abstract κλάσεις ή τις συναρτήσεις με το keyword open. 

abstract class Polygon {
    abstract fun draw()
}

class Rectangle : Polygon() {
    override fun draw() {
        // Σχεδιάζει το ορθογώνιο (rectangle)
    }
}

Μπορούμε να υπερκαλύψουμε (override) ένα non-abstract open μέλος με ένα abstract. 

open class Polygon {
    open fun draw() {
        // Κάποια default μέθοδος σχεδιασμού πολυγώνων
    }
}

abstract class WildShape : Polygon() {
    /* Κλάσεις που κληρονομούν την WildShape χρειάζεται να παρέχουν τη δική τους
    μέθοδο σχεδίασης αντί να χρησιμοποιήσουν τη default μέθοδο της κλάσης Polygon*/
    abstract override fun draw()
}

Companion objects

Αν χρειαζόμαστε να γράψουμε μία συνάρτηση που θα μπορεί να κληθεί χωρίς να έχουμε στιγμιότυπο κλάσης αλλά αυτό θα χρειάζεται να προσπελαστεί στο εσωτερικό μίας κλάσης (όπως μία factory method), μπορούμε να τη γράψουμε ως μέλος ενός object declaration μέσα σε αυτήν την κλάση.

Ακόμα πιο συγκεκριμένα, αν δηλώσουμε ένα companion object μέσα στην κλάση μας, μπορούμε να προσπελάσουμε τα μέλη της χρησιμοποιώντας μόνο το όνομα της κλάσης ως qualifier.

Πρωτεύων vs. Δευτερεύοντος κατασκευαστή

Η Kotlin υποστηρίζει την προδιαγραφή του πρωτεύοντος κατασκευαστή (primary constructor) ως τμήματος του ίδιου του ορισμού της κλάσης, αποτελούμενου από μία λίστα ορισμάτων που ακολουθούν το όνομα της κλάσης. Αυτή η λίστα ορισμάτων υποστηρίζει μία εκτεταμμένη σύνταξη στις λίστες ορισμάτων συναρτήσεων της Kotlin, που επιτρέπει τη δήλωση ιδιοτήτων της κλάσης στον πρωτεύοντα κατασκευαστή, συμπεριλαμβανομένων των ιδιοτήτων της ορατότητας (visibility), επεκτασιμότητας (extensibility) και μεταβλητότητας (mutability). Επιπρόσθετα, όταν καθορίζεται μία υποκλάση, οι ιδιότητες στα super-interfaces και τις super-classes μπορούν να υπερκαλυφθούν (overridden) στον πρωτεύοντα κατασκευαστή. 

// Παράδειγμα κλάσης που χρησιμοποιεί πρωτεύοντα κατασκευαστή
// (Απαιτείται μόνον ένας κατασκευαστής για αυτήν την κλάση)
open class BaseUser(open var isSubscribed: Boolean)
open class PowerUser(protected val nickname: String, final override var isSubscribed: Boolean = true):BaseUser(isSubscribed) { }

Παρόλα αυτά, σε περιπτώσεις όπου περισσότεροι από ένα κατασκευαστές χρειάζονται σε μία κλάση, μπορεί να χρησιμοποιηθεί ένας πιο γενικός κατασκευαστής που ονομάζεται σύνταξη δευτερεύοντος κατασκευαστή (secondary constructor syntax) που προσομοιάζει με την σύνταξη κατασκευαστών που χρησιμοποιούνται σε γλώσσες αντικειμενοστραφούς προγραμματισμού (object-oriented languages) όπως οι C++, C#, και Java. 

// Παράδειγμα κλάσης που χρησιμοποιεί σύνταξη δευτερεύοντος κατασκευαστή
// (Περισσότεροι του ενός κατασκευαστές απαιτούνται για αυτήν τν κλάση)
class Context
class AttributeSet
open class View(ctx:Context) {
    constructor(ctx: Context, attr: AttributeSet): this(ctx)
}
class MyButton : View {
    // Κατασκευαστής #1 
    constructor(ctx: Context) : super(ctx) { 
    } 
    // Κατασκευαστής #2
    constructor(ctx: Context, attr: AttributeSet) : super(ctx, attr) {
        // ... 
    }
}
